# 🔍 AI 合同审查助手 v2.0

> 专为非技术人员设计的「AI 智能审查插件」。不用懂写代码，不用配 Python，双击就能在 **Microsoft Word** 或 **WPS Office** 里获得一个 AI 法律助手。

**✅ 同时支持 Microsoft Word 和 WPS Office，Windows 和 macOS 均可使用，下载一份，全平台通用。**

---

## ✨ 核心能力

本插件以原生侧边栏（Taskpane）的形式运行，为您提供沉浸式的 AI 审阅体验：

- 🖥️ **全平台兼容**：支持 **Microsoft Word**（Windows / macOS）和 **WPS Office**（Windows / macOS），同一份安装包，启动时自动识别您打开的是哪个软件，无需额外配置。
- 📑 **一键交叉审查**：从预先设定的四维审查标准（主要风险、权利义务不对等、关键条款缺失、格式规范）出发，秒级出具审查结构报告。
- 🎭 **审查视角切换**：你是甲方还是乙方？一键设定审查立场，AI 将聪明地调整屁股，专为你挑出不利于己方的隐藏陷阱。
- ⚖️ **原文精准定位与批注**：不仅告诉你哪里有风险，更能直接在文档正文中高亮风险段落，并生成标准审阅批注。
- 🤖 **划选对话陪伴**：遇到不懂的条款？直接在文档里用鼠标选中那段话，右侧 AI 就会针对你的划选内容进行解答、翻译或改写。
- 📝 **自定义审查模板**：支持用户随时按具体场景（如租赁、劳动、投资）自行设计和加载专属的"提示词模板"，把你的审查经验固化给 AI。
- 🔒 **进阶离线断网审查**：支持接入本地大模型（Ollama），彻底断网离线运算，极致保障商业机密安全。
- 🔗 **外挂法律检索 (MCP)**：支持连接外部专业法律数据库，查法条、核对法条引用有效性，告别 AI 法律幻觉。

## 🎯 你的电脑需要准备什么？

为了让这个助手在你的电脑上跑起来，你需要安装 **1 个前置软件**（叫做 Node.js，它负责给助手提供运行环境）。

> 💡 **为什么要装这个？** 因为苹果和微软限制了直接安装未上架应用商店的插件，我们需要用开发者的身份把它"挂载"到 Word/WPS 里。这就需要 Node.js 环境。

### 第一步：安装 Node.js（只需做一次）

1. 点击打开官网：[Node.js 官方下载页](https://nodejs.org/zh-cn)
2. 选择 **LTS（长期支持版）** 下载（会有个绿色的按钮推荐你点）
3. 下载完成后，双击安装包。
4. **一路点击"下一步 (Next)"和"同意 (Agree)"**。什么选项都不要改，直到安装完成。

> ✅ **检查是否成功**：
>
> - **Windows 用户**：按键盘上的 `Win + R` 键，输入 `cmd` 按回车。在跳出来的黑框框里输入 `node -v`。如果出现 `v20...` 这样的字样，说明成功。
> - **Mac 用户**：打开"终端"（启动台里搜索"终端"）。输入 `node -v`，出现版本号即为成功。

---

## 🚀 第二步：下载插件并启动

完成第一步后，一切就非常简单了。

1. **下载本项目**：在这个网页的右上方找到绿色的 **`<> Code`** 按钮，点击后选择 **`Download ZIP`**。
2. 下载后，**解压缩**这个 ZIP 文件，把它存放在一个你找得到的地方（比如桌面）。
3. 打开解压后的文件夹。

### 🪟 对于 Windows 用户

- 双击文件夹里的 **`启动.bat`** 文件。
- *（如果一闪而过退出了，右键点击它 → 选"以管理员身份运行"）*
- 脚本会询问你使用的是哪个软件：

  ```text
  [W]  Microsoft Word（默认）
  [P]  WPS Office
  ```

  输入 `W` 或 `P` 后按回车，脚本会自动完成对应的配置。

### 🍎 对于 Mac（苹果电脑）用户

1. 打开 **终端**（启动台里搜索"终端"）。
2. 输入 `bash`（注意 `bash` 后面带一个空格），然后把文件夹里的 **`启动.command`** 文件**拖进**终端窗口里，按回车运行。
3. 同样会询问 `W`（Word）还是 `P`（WPS），输入后按回车即可。
4. 中途可能会弹窗要求输入**开机密码**（这是为了安装安全证书，允许 Word 和助手通信；WPS 模式不需要此步骤）。

> 💡 **首次自愈特性**：第一次通过上述步骤运行后，脚本会自动自我修复 Mac 权限拦截，**以后使用时就可以直接双击 `启动.command` 运行了**！

> ⚠️ **全系统必看警告**
> 启动成功后，**绝对不要关闭那个黑框框（终端）！**
> 黑框框就是助手的大脑，关掉黑框框，插件就会"断联"。不用的时候最小化即可。

## 📝 第三步：在 Word 或 WPS 中调出助手

### 在 Microsoft Word 中调出（Office 365 / 2016+）

1. 启动脚本（`启动.bat` / `启动.command`）选择 **`W`**，插件会自动注册并弹出一份 Word 文档。
2. 在 Word 顶部菜单找到 **【AI 合同审查助手】** 按钮点击即可。
3. **找不到按钮怎么办？**
   由于不同 Word 版本界面差异，如果顶部的【开始】选项卡里没看见按钮，请点击上方菜单的 **【插入】** → 找到并点击 **【我的加载项】**（或者就叫【加载项】）→ 在弹出的窗口里双击列表中的 **"AI 合同审查助手"** 即可调出。

> ⚠️ 首次使用 Word 时，若面板白屏显示"证书不受信任"，请参考下方「常见问题」第 1 条处理。

### 在 WPS Office 中调出

1. 启动脚本选择 **`P`**，脚本会自动完成所有配置（首次会多花约 30 秒安装工具）。
2. **完全退出 WPS 再重新打开**（第一次配置后需要重启 WPS 才能生效）。
3. WPS 顶部工具栏出现新的 **【智能审查】** 选项卡，点击 **【打开审查面板】** 即可。

> 💡 重启一次之后，以后每次直接双击启动脚本选 `P`，WPS 打开就能用，不需要额外操作。

## 💥 常见问题排雷 (FAQ)

### 1. 助手面板白屏，显示什么"证书不受信任 / 无法加载"？

这是 Word 插件的安全机制导致的。**仅 Microsoft Word 用户需要处理此问题，WPS 用户不受影响。**

**怎么修？**
打开你电脑里的浏览器（Chrome/Edge），把这个网址输进去：`https://localhost:3000`
浏览器会跳出一个警告页面（说连接不安全），点击 **【高级】** → 选 **【继续前往 localhost (不安全)】**。
看到页面有内容后，关掉浏览器，去 Word 里重新点开插件就好了。

### 2. WPS 用户点击"智能审查"后侧边栏是完全一片空白？

如果你是**直接下载的 ZIP 压缩包**，并且在使用 WPS 时遇到白屏，这通常是因为两个原因：
  
**原因一：你没有执行打包构建（新版脚本已自动处理）**
WPS 需要读取真实的物理文件。在旧版本中，你需要先执行 `pnpm run build`。现在最新的启动脚本已经会**自动检测并构建**，遇到此问题请确保脚本正常跑完了构建流程。

**原因二：本地证书被拦截（最常见）**
如果已经 build 过了还是白屏，说明你的 WPS 拦截了本地测试的临时 HTTPS 证书（它觉得不安全）。
**怎么修？**
保持 `启动.bat` 的黑框运行着，打开你的 Chrome 或 Edge 浏览器，访问：`https://localhost:3000/taskpane-wps.html`
浏览器会弹出红色的安全警告。点击底部的 **【高级】** -> **【继续访问 localhost (不安全)】**。
只要浏览器里成功看到了白色的界面，你的电脑就信任这个证书了。现在关掉浏览器，回到 WPS 重新打开侧边栏，界面就会瞬间出现。

### 3. 【Mac 专属】通过 bash 运行依然说权限不足？或者隔离属性报错？

如果有极小概率自动修复失效：

- **手动赋权**：打开终端，输入 `chmod +x` (注意有空格)，把 `启动.command` 拖进去回车。
- **手动解隔离**：打开终端，输入 `xattr -rd com.apple.quarantine` (注意有空格)，把**整个项目文件夹**拖进去回车。执行后即可双击运行。

### 4. 【Mac 专属】终端大量红字报错 `EACCES` 或者 `mkdir` 权限被拒？

如果脚本安装 pnpm 失败，说明你这台电脑曾经用不规范的方式安装过 Node，导致系统目录被锁定。

- **终极解决**：在终端里手动输入并执行 `curl -fsSL https://get.pnpm.io/install.sh | sh -` 以独立安装包管理器，然后重新运行启动脚本。

### 5. 【Mac 专属】脚本里选了 Word，但最终弹出来的却是 WPS？

这是由于 macOS 的 "文件关联默认打开机制" 导致的：你的电脑认为所有 `.docx` 都该给 WPS 打开。

- **修改关联**：随便找一个桌面上的 `.docx` 文件，**右键**点击它 -> 按住键盘的 **`Option (Alt)`** 键不放 -> 点击 **【始终以此方式打开】** -> 在弹出的应用列表里点选 **Microsoft Word**。然后再次双击运行脚本即可。

### 6. 一点击「开始审查」，就报错 401？

这代表你的"钥匙（API Key）"填错了，或者你的 AI 账号里没有钱（欠费了）。请去对应官网查下余额。

### 7. Word 版和 WPS 版需要分开下载吗？

**不需要。** 两个平台用的是同一份代码和同一个启动脚本。下载本仓库后，程序会在启动时自动识别你使用的是 Word 还是 WPS，并切换到对应的适配器。你只需按照第三步里对应软件的说明操作即可。

### 8. 数据安全吗？会泄露给网上吗？

你的合同内容**只会在你点击"审查"的那一瞬间**，发送给你在设置里填写的"那个 AI 厂商（比如深度求索、阿里）"。代码里**没有任何后门或截留**。如果实在对保密要求极高，请使用下方的**本地离线方案**。

## 💻 进阶教程：如何利用 Ollama 做到完全离线断网审查（绝密安全）

如果是审查极端机密的商业合同，你不想把只言片语发送到任何一家互联网公司的服务器上，你可以把 AI 模型**免费下载到你自己的电脑主机里**！

> ⚠️ 这个方案要求你的电脑足够强大（推荐拥有大于 16GB 大显存的独立显卡，或者 32GB 以上统一内存的 Mac 系列）。

### 第一步：下载控制台 Ollama

1. 打开 [Ollama 官方网站](https://ollama.com/)
2. 点击中间巨大的 **Download** 按钮下载并安装到电脑。
3. 安装完成后，Ollama 图标会出现在电脑右下角（或 Mac 状态栏）的小托盘里。

### 第二步：下载国产大模型本体装入电脑

1. 打开电脑的命令行（Windows 按 `Win+R` 输 `cmd`，Mac 搜 `终端`）。
2. 输入以下命令并按回车（大约数 GB 大小，需要等一会儿）：

```bash
ollama run qwen3.5:14b
```

*(注：如果你的电脑显存有 24GB，可以改用更聪明的 `qwen3.5:32b`；如果你只是普通轻薄本没有独立显卡，可以降级用 `qwen3.5:7b`。)*

1. 等它跑完并显示 `>>> Send a message` 时，说明下载安装成功。在框里输入 `/bye` 回车即可退出。

### 第三步：回到插件里连接

1. 打开你的 Word/WPS 和「智能审查助手」。
2. 切到 **设置** 页面。
3. 提供商选：**Ollama**
4. 模型名称填入你刚才装的那个名字，例如：`qwen3.5:14b`。
5. （API 地址不需要填，API Key 不需要填）。

现在你可以**拔断网线，断开 WiFi** 了！点击开始审查，一切都在你的本地运算，全世界没有人能拿到这份合同的数据。

## 🔗 高阶玩法：给 AI 外挂权威法律数据库 (MCP)

如果你觉得 AI 有时候会"胡编乱造"法条（即 AI 幻觉），你可以通过最新的 **MCP (Model Context Protocol)** 技术，给它外挂一个真正的、拥有标准法条数据的外脑。

这里强烈推荐搭配使用专为法律场景打造的开源数据库：**[Legal Database](https://github.com/sublatesublate-design/legal-database)**

### 什么是 Legal Database？

这是一个同样开源的本地法律检索服务器。把它和「智能审查助手」连接后，你的 AI 将获得以下超能力：

1. **真实法条检索**：AI 在审查合同时，可以随时去数据库里翻阅真实的《民法典》、《公司法》原文，基于精准法条出具意见。
2. **法条引用核对**：AI 可以帮你一键扫描整份合同，找出里面引用的法条是不是过期了（比如是不是还在引用已经废止的《合同法》），并给出最新有效的替换建议。
3. **法律状态检查**：确保提供的法律意见是有现行法律支撑的。

### 如何连接？（非常简单）

1. 参考 [Legal Database 项目主页](https://github.com/sublatesublate-design/legal-database) 的指南，把你电脑上的法律知识库跑起来。
2. 记下那个数据库运行的地址（通常是类似 `http://localhost:8000/sse` 这样的网页链接）。
3. 打开「智能审查助手」，翻到 **设置** 页面。
4. 往下划，找到 **【MCP 工具扩展】** 区域。
5. 点击【添加】，在 URL 处填入那个地址：`http://localhost:8000/sse`，传输方式选 SSE。
6. 点击 **添加** 确认。

连接成功后，对应服务器会亮起绿灯。下次让它审查合同时，它就会**自动调用**这个本地数据库，给你绝对权威、100% 准确的法律解答！
